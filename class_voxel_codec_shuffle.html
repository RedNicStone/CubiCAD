this class represents a shuffled voxel codec, stored like this: n data 1 [ <attribute0>, <attribute1>, ..., <attributeN>, 2 <attribute0>, <attribute1>, ..., <attributeN>, 3 <attribute0>, <attribute1>, ..., <attributeN> ]

this implementation uses just as much memory as "VoxelCodecLinear", but since the attributes are close together it should be faster to work on
